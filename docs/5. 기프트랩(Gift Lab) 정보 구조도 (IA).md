## **1\. 개요 (Overview)**

본 문서는 **기프트랩(Gift Lab)** 앱의 메뉴 구조와 각 화면에 포함될 정보의 위계를 정의한다. 불필요한 Depth를 최소화하여 사용자가 \*\*'입력 → 결과'\*\*라는 핵심 가치에 최단 경로로 도달하도록 설계되었다.

## **2\. IA 구조도 (Structure Map)**

\[ROOT\] 기프트랩 (Gift Lab)  
├── 1\. 스플래시 (Splash)  
│   └── 로고 및 자동 로그인  
├── 2\. 메인: 선물 의뢰 (Main / Input)  
│   ├── Step 1: 대상 선택 (Target)  
│   ├── Step 2: 기념일 선택 (Event)  
│   ├── Step 3: 성격/MBTI 입력 (Persona)  
│   ├── Step 4: 예산 설정 (Budget)  
│   └── Step 5: 옵션/제외 설정 (Option)  
├── 3\. 분석 및 광고 (Processing)  
│   ├── 로딩 애니메이션 (Loading)  
│   └── 전면 광고 (Interstitial Ad)  
├── 4\. 결과 리포트 (Result)  
│   ├── 추천 카드 리스트 (Curated List)  
│   ├── 하단 배너 광고 (Banner Ad)  
│   └── 액션 메뉴 (Share/Retry)  
└── 5\. 설정 및 정보 (Settings)  
    └── 약관 및 버전 정보

## **3\. 화면별 상세 구성 (Detailed Description)**

### **1.0 스플래시 (Splash)**

앱 실행 시 최초로 노출되는 진입 화면.

* **UI 요소:**  
  * 앱 로고 (Lottie Animation)  
  * 슬로건: "당신의 센스를 연구합니다"  
* **Data/Action:**  
  * (Back) Supabase Auth: 익명 로그인 시도 및 세션 체크.  
  * (Action) 2초 후 2.0 메인으로 자동 전환.

### **2.0 메인: 선물 의뢰 (Main Input)**

단일 페이지 내에서 PageView로 전환되는 5단계 입력 폼.

#### **2.1 Step 1: 대상 (Target)**

* **UI 요소:**  
  * 진행 표시줄 (Progress Bar: 20%)  
  * 질문 텍스트: "누구에게 선물하시나요?"  
  * 관계 선택 버튼 그리드 (연인, 부모님, 친구, 직장동료, 기타)  
  * 성별 선택 토글 (남/여)  
  * 연령대 선택 휠/피커 (10대\~60대+)  
* **Data:** relation (String), gender (String), age\_group (String)  
* **Interaction:** 버튼 선택 시 자동으로 다음 스텝 스크롤.

#### **2.2 Step 2: 기념일 (Event)**

* **UI 요소:**  
  * 진행 표시줄 (Progress Bar: 40%)  
  * 질문 텍스트: "어떤 특별한 날인가요?"  
  * 상황 선택 칩 (생일, 집들이, 기념일, 합격/승진, 사과, 가벼운 선물, 고백)  
* **Data:** event\_type (String)

#### **2.3 Step 3: 성격/MBTI (Persona)**

* **UI 요소:**  
  * 진행 표시줄 (Progress Bar: 60%)  
  * 질문 텍스트: "그 분은 어떤 성향인가요?"  
  * MBTI 토글 스위치 (4개 행)  
    * E (외향) vs I (내향)  
    * N (직관) vs S (현실)  
    * F (감성) vs T (이성)  
    * P (즉흥) vs J (계획)  
  * (Optional) '잘 모르겠어요' 체크박스 → 클릭 시 '성격 키워드 태그' 선택 화면으로 대체.  
  * **실시간 요약 문구:** (예: "활기차고 현실적인 리더 스타일이네요\!")  
* **Data:** mbti\_code (String), keywords (List\<String\>)

#### **2.4 Step 4: 예산 (Budget)**

* **UI 요소:**  
  * 진행 표시줄 (Progress Bar: 80%)  
  * 질문 텍스트: "예산은 어느 정도 생각하세요?"  
  * 가격 범위 슬라이더 (Slider)  
  * 현재 금액 표시 텍스트 (예: "5만원 \~ 10만원")  
* **Data:** min\_price (Int), max\_price (Int)

#### **2.5 Step 5: 옵션 및 완료 (Option)**

* **UI 요소:**  
  * 진행 표시줄 (Progress Bar: 100%)  
  * 질문 텍스트: "혹시 피해야 할 선물이 있나요?"  
  * 제외 키워드 입력 필드 (Text Input) \- (예: "꽃, 먹을거 제외")  
  * $$CTA$$  
    결과 분석 요청하기 버튼 (Floating Action Button)  
* **Data:** excluded\_items (String)  
* **Interaction:** 버튼 클릭 시 유효성 검사 후 3.0 분석 화면 전환.

### **3.0 분석 및 광고 (Processing)**

데이터 처리와 수익화가 동시에 일어나는 중간 화면.

#### **3.1 로딩 (Loading)**

* **UI 요소:**  
  * 연구소 컨셉 애니메이션 (Loop)  
  * 동적 문구: "{관계}님을 위한 {예산}대 선물을 찾고 있어요..."  
* **Data:** AI API 호출 상태, 광고 로드 상태.

#### **3.2 전면 광고 (Interstitial Ad)**

* **UI 요소:**  
  * Google AdMob 전면 광고 오버레이.  
  * 닫기(X) 버튼 (광고 SDK 제공).  
* **Interaction:** 닫기 버튼 클릭 시 4.0 결과 화면 진입.

### **4.0 결과 리포트 (Result)**

AI가 생성한 최종 추천 목록 화면.

#### **4.1 상단 및 네비게이션**

* **UI 요소:**  
  * 타이틀: "연구 결과 리포트"  
  * (Right) 설정 아이콘 버튼.

#### **4.2 추천 카드 리스트 (Card List)**

Swiper 형태의 카드 UI. (3\~5개 생성)

* **UI 요소 (개별 카드):**  
  * 상품 이미지 (플레이스홀더 아이콘 or 카테고리 이미지)  
  * **핵심:** 상품명 (Product Name)  
  * **핵심:** 추천 이유 소견서 (AI Generated Text)  
  * 예상 가격대  
  * $$버튼$$  
    **"쿠팡 최저가 확인하기"**  
* **Data:** recommendation\_list (List\<JSON\>)  
* **Interaction:** 버튼 클릭 시 외부 브라우저/앱 실행 (DeepLink).

#### **4.3 하단 영역 (Footer)**

* **UI 요소:**  
  * $$Ad$$  
    하단 배너 광고 (Banner Ad)  
  * 버튼: "친구에게 리스트 공유하기" (Kakao Share)  
  * 버튼: "다른 선물 찾기" (Reset)  
* **Data:** 법적 고지 문구 ("쿠팡 파트너스 활동...")

### **5.0 설정 (Settings)**

Modal Bottom Sheet 형태로 제공.

* **UI 요소:**  
  * 버전 정보 (v1.0.0)  
  * 이용약관 보기 (Webview Link)  
  * 개인정보처리방침 (Webview Link)  
  * 문의하기 (Email Link)
