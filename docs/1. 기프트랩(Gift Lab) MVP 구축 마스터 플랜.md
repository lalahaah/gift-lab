# **1\. 기프트랩(Gift Lab) MVP 구축 마스터 플랜**

## **Phase 1: 비즈니스 로직 및 기획 (Blueprinting)**

가장 중요한 단계입니다. 코드를 짜기 전에 '어떤 데이터를 넣어서 어떤 결과를 뽑을지' 명확히 정의합니다.

* \[ \] **1-1. 핵심 페르소나 정의 (Input Logic)**  
  * \[ \] 5단계 입력 변수 확정 (1. 대상/관계 2\. 기념일 종류 3\. MBTI/성격 키워드 4\. 예산 범위 5\. 제외할 선물/과거 이력)  
* \[ \] **1-2. AI 큐레이터 프롬프트 설계 (Output Logic)**  
  * \[ \] '수석 연구원' 페르소나를 반영한 시스템 프롬프트(System Prompt) 작성  
  * \[ \] AI 응답 JSON 스키마 정의 (상품명, 추천 이유(3줄), 검색 키워드, 카테고리)  
* \[ \] **1-3. 수익화 흐름 설계 (Monetization Flow)**  
  * \[ \] AdMob 삽입 위치 확정 (로딩 화면 전면 광고, 결과 화면 배너/네이티브 광고)  
  * \[ \] 쿠팡 파트너스/아마존 어소시에이트 링크 변환 로직 설계 (딥링크 구조)  
* \[ \] **1-4. 화면 흐름도(User Flow) 스케치**  
  * \[ \] 스플래시(로고) \-\> 온보딩(5단계 질문) \-\> 분석 로딩(광고) \-\> 결과 리스트 \-\> 상세/구매 이동

## **Phase 2: 테크 셋업 & DB 설계 (Foundation)**

Flutter와 Supabase를 연동하고 개발 환경을 구축합니다.

* \[ \] **2-1. 프로젝트 초기화**  
  * \[ \] Flutter 프로젝트 생성 (flutter create gift\_lab)  
  * \[ \] Supabase 프로젝트 생성 및 연동  
  * \[ \] GitHub 레포지토리 생성 및 연동  
* \[ \] **2-2. Supabase DB 스키마 구축**  
  * \[ \] profiles 테이블: 사용자 기본 정보 (UUID, 기기정보, 선호 언어)  
  * \[ \] gift\_queries 테이블: 사용자 입력값 로그 (성격, 예산 등 \- 추후 데이터 분석용)  
  * \[ \] recommendations 테이블: AI가 생성한 추천 결과 저장 (공유/저장 기능용)  
  * \[ \] clicks 테이블: 사용자가 실제로 클릭한 상품 로깅 (CTR 분석용)  
* \[ \] **2-3. Edge Function (AI 연동) 설정**  
  * \[ \] Supabase Edge Function 환경 설정 (Deno)  
  * \[ \] OpenAI (또는 Gemini) API 키 발급 및 환경변수 설정  
  * \[ \] 입력값(JSON)을 받아 AI 추천 결과(JSON)를 반환하는 기본 함수 작성

## **Phase 3: 프론트엔드 개발 (MVP Implementation)**

핵심 기능 구현에 집중합니다. 디자인은 깔끔함(Clean)을 우선합니다.

* \[ \] **3-1. 공통 UI/UX 개발**  
  * \[ \] 기프트랩 테마(컬러 팔레트, 폰트) 적용  
  * \[ \] 스플래시 스크린 구현 (로고 애니메이션)  
* \[ \] **3-2. 입력 폼(Input Form) 개발**  
  * \[ \] Step-by-Step 입력 위젯 구현 (PageView 또는 Stepper 활용)  
  * \[ \] 각 단계별 데이터 State Management 처리 (Riverpod/GetX/Provider 중 택1)  
* \[ \] **3-3. 로딩 및 결과 페이지 개발**  
  * \[ \] 로딩 화면: Lottie 애니메이션 적용 \+ AdMob 전면 광고 로드 로직  
  * \[ \] 결과 화면: 추천 카드 리스트 UI 구현  
  * \[ \] '구매하러 가기' 버튼 클릭 시 쿠팡 검색/상품 페이지로 딥링크 연결 기능

## **Phase 4: 수익화 및 고도화 (Monetization & Polish)**

돈을 벌기 위한 장치를 연결하고 앱의 완성도를 높입니다.

* \[ \] **4-1. 광고 연동 (AdMob)**  
  * \[ \] Google AdMob 계정 생성 및 앱 ID 발급  
  * \[ \] 테스트 광고 ID로 전면 광고(Interstitial) 및 배너 광고 작동 확인  
* \[ \] **4-2. 제휴 마케팅 링크 로직**  
  * \[ \] AI가 추출한 '검색 키워드'를 쿠팡 파트너스 검색 URL로 동적 변환하는 함수 구현  
  * \[ \] (선택) 쿠팡 API 승인 전이라면 'Web Search' 링크로 대체하는 폴백(Fallback) 로직  
* \[ \] **4-3. 다국어 처리 (Localization)**  
  * \[ \] 한국어(KR) / 영어(EN) 리소스 파일 분리  
  * \[ \] 기기 설정에 따라 언어 및 연결 쇼핑몰(쿠팡 vs 아마존) 자동 전환

## **Phase 5: 테스트 및 출시 (Launch)**

시장에 내놓기 위한 마지막 점검입니다.

* \[ \] **5-1. 최종 테스트**  
  * \[ \] 다양한 시나리오(예산 부족, 이상한 입력 등)에 대한 AI 응답 예외 처리  
  * \[ \] 광고 정책 위반 여부 확인 (클릭 유도 금지 등)  
* \[ \] **5-2. 스토어 등록 준비**  
  * \[ \] 앱 아이콘 및 스토어 스크린샷 제작 (Canva 등 활용)  
  * \[ \] 개인정보 처리방침 문서 작성 (무료 툴 활용)  
  * \[ \] 구글 플레이 콘솔 / 애플 앱스토어 커넥트 계정 생성 및 앱 등록

## **Phase 6: 마케팅 및 운영 (Growth)**

출시 후 데이터를 보며 개선합니다.

* \[ \] **6-1. 초기 마케팅**  
  * \[ \] 숏폼(릴스/쇼츠) 콘텐츠 제작: "최악의 선물 vs 최고의 선물"  
  * \[ \] 커뮤니티 바이럴 (MBTI 관련 카페 등)  
* \[ \] **6-2. 데이터 분석 및 업데이트**  
  * \[ \] Supabase 로그 분석: 어떤 MBTI/기념일 검색이 많은지 파악  
  * \[ \] 유저 피드백 반영하여 프롬프트 튜닝 (V1.1 업데이트)
